# レスポンシブ対応の確認手順

各画面のレスポンシブ対応実装が完了しました。以下の手順で動作確認を行ってください。

## 実装内容
1.  **ゲーム画面 (GameScreen)**
    - **横画面対応**: 画面を回転させると、自動的に「盤面（左）」と「操作ボタン（右）」の並列レイアウトに切り替わります。
    - **スクロール対応**: 縦画面でも画面サイズが小さい場合、全体がスクロール可能になり、ボタンの下部が見切れる問題が解消されています。
    - **タブレット制限**: 画面幅が広い場合でも、盤面が過度に大きくならないよう最大幅制限（600px）を設けました。

2.  **タイトル画面 (TitleScreen)**
    - **タイトル文字**: 画面幅が狭い場合、タイトルロゴが自動的に縮小され、改行が崩れるのを防ぎます。
    - **縦方向の間隔**: 固定値ではなく画面の高さに対する比率（%）で間隔を調整するため、縦に長い/短い端末でもバランスよく表示されます。

## 検証項目

### 1. ゲーム画面の横向き (Landscape)
1.  任意の難易度でゲームを開始します。
2.  端末を横向きに回転させます（またはシミュレータで回転）。
3.  **確認点**:
    - 左側に数独の盤面、右側に数字パッドとボタンが表示されること。
    - 盤面が画面内に収まり、操作不能な領域がないこと。

### 2. ゲーム画面の縦向き・小画面 (Portrait)
1.  縦向きでゲームを表示します。
2.  （可能であれば）画面分割機能などで表示領域を狭くします。
3.  **確認点**:
    - 画面を上下にスクロールでき、一番下のボタンまでアクセスできること。

### 3. タイトル画面
1.  タイトル画面を表示します。
2.  **確認点**:
    - 極端に幅の狭い端末でも「おしゃべりクッキーの SUDOKU」の文字が見切れたり崩れたりしていないこと（文字サイズが小さくなって収まっていること）。

## 既知の制限事項
- 極端に横長の画面（アスペクト比 21:9 など）では、左右の余白が大きくなる場合がありますが、操作には影響ありません。
